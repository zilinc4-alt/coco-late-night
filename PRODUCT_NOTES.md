## 一、产品定位（一句话）

**"假装点外卖"的情绪疗愈站。** 不产生真实交易、不下发订单、不消费卡路里，只让你走完一次"想点外卖"的完整心理流程，用**等待稀释冲动**。

- 首页大字：**今晚想点外卖吗？**
- 副标：*想点，但又不想真的点？先在这里馋一下，等冲动过去。*
- 主 CTA：`开始点单` / 次 CTA：`我是店家`
- 底部脚注：*不会花钱，不会下单，不会真的送来。但你可以完整体验一次"想点外卖"的过程。*
- 深色 + 橙金渐变，"宵夜"主题，深夜氛围拉满

### 1. 首页（`#/home`）

### 2. 店家列表（`#/restaurants/bbq` 等）
- 面包屑 "附近夜宵" + 标题 "附近这些店还开着" + "先选分类，再挑一家进店看看。"
- **分类横滑标签**：8 个（emoji+文字），当前 tab 高亮

  - 左封面图 + 店名 + `28-42 分钟`（预计送达）

  - 3 条标签（"今晚免配送费"、"炭火很到位"、"打烊前最后一单"）
- 分类结束后有分隔线 "以下为其他类别店铺"，继续列其他店（引导跨分类逛）

### 3. 店铺菜单页（`#/menu/late-bbq-lab`）
- 店铺头部：店名 + 评分/月售/时间 + 3 条标签 + 一句店铺描述（"炭火味很足，适合奖励今天撑到现在的自己。"）
- 菜品按小分类分组：招牌 / 素一点 / 加一点 / 套餐 / 饮品 / 主食（不同店可能不同）
- 菜品卡：图 + 名字 + **情绪化文案**（灵魂）+ `月售 507` + `¥9` + `+`
  - 文案例："开完会还没被夸，先给自己一点香气"、"把今天的硬撑，先串成一口奖励"、"通勤到家的那口松弛感，先放这里"
- 底部粘性购物车条：🛒 数量 + 金额 + "还差 ¥25 起送" + `去结算` + "今晚不点券可用"

### 4. 确认订单（`#/checkout`）
- 标题："确认今晚这单"，副标"不用填写真实地址，系统已为你准备好今晚不点券。"
- **订单卡**：店名 + 菜品 × 数量 + 单项金额，合计
- **配送信息**：
  - 地址下拉（预置多个"梗地址"：
    - 我的深夜情绪收件处
    - 明早不会后悔小区
    - 被窝 3 号楼
    - 理智恢复中心）
  - 支付方式：**今晚不点券**（固定）
  - 预计送达：`12-20 分钟后`
  - 订单备注：`少放后悔，多放一点清醒`
- 底部提示条：`预计省下 ¥72  ·  大约少摄入 2480-2880 千卡`
- 主 CTA：`确认下单`

### 5. 配送等待（`#/waiting`）——**产品灵魂**
- 顶部：`订单 #30822`
- H1：**骑手正在赶来**
- 描述："你的订单已经生成。现在最重要的不是吃到它，而是等一等。"
- **倒计时大卡**：`预计 14 分 49 秒 后送达`
- 骑手信息：`骑手：小冷静` / `小理智`（随机 nickname）+ 进度百分比（0%→100%）
- **格子地图**（约 8×8 网格）：右上"你"图钉，左侧店位置，橙色小摩托 🛵 沿虚线从店往"你"移动，进度绑倒计时
- 三个转移注意力按钮：
  - `我先去喝杯水` → toast "好，去喝两口水。骑手会继续在路上。"
  - `我去刷个牙` → "刷牙是一个很强的暂停键。订单还会继续等你。"
  - `我还是很想点` → 弹一段安抚文案："没关系，冲动还在很正常。再等 3 分钟，还是想点，你再决定…" + `继续等一会儿` 按钮（这条路径也不会真下单，就是一直等）
- 底部小字："这单不会产生真实配送，你只需要陪这个冲动待一会儿。"

**产品心机**：倒计时故意长（12-20 分钟），过程中不断安抚用户，等冲动过去用户自然离开——**永远不会到达"送达"页面**。这是有意的：让"想点"这件事不再急促。

## 四、店家端完整流程

### 1. 店家入口（`#/merchant`）
- 标题 "我是店家" + 说明 "开一家只负责让人馋的小店。第一版不做登录，也没有真实经营、收款或配送。小店会保存在服务器上，**店铺口令**就是你的找回凭证。"
- 两个卡片：
  - **新店家**：说明 + `开一家不会真的出餐的小店` 按钮
  - **找回已有小店**：输入 `店铺口令`（例：`NIGHT-ABC123`）+ `找回小店` 按钮

### 2. 开店表单（`#/merchant/new`）
- 顶部有一条粘性状态条 "有一单夜宵正在路上 14 分 12 秒"（如果之前下过单还在等）
- 表单字段：
  - `店铺名称`（例："加班后烧烤档"）
  - `店铺分类`（下拉，9 个 emoji+名）
  - `店铺介绍`（例："专治下班后那口空落落。"）
  - `店铺封面`（图片上传，说明"会压缩后随店铺资料同步到服务器"）
- 主 CTA：`保存并开店`

### 3. 店家后台（`#/merchant/dashboard`）
- 顶部 toast："小店开好了，口令已经同步到服务器。"
- 说明："这家店只营业在想象里。店铺资料和菜单会用口令同步到服务器。"
- 状态："正在同步服务器订单和访问数据..."
- **店铺卡**：封面 + 名字 + `烧烤 · 测试店铺2` + **`店铺口令：NIGHT-ULYE3N`** + 4 个操作按钮：
  - `编辑店铺信息` / `复制店铺口令` / `复制店铺地址` / `预览我的店铺`
- **今日营业数据** 卡（4 格）：
  - `今日访问`：0（今日有 0 人路过你的小店）
  - `今日下单`：0（今天有 0 单不会真的送达）
  - `今日幻想收入`：¥0（虚拟收入，不会到账）
  - `今日被馋`：0（你的菜今天被馋了 0 次）
  - 右上有 `查看完整数据`
  - "今天还在等第一位客人"（无数据兜底）
- **菜品管理**：
  - "先上新一道菜" 引导
  - 主 CTA：`新增一道不会真的送来的菜`
  - 兜底："菜单还是空的。普通用户进店后需要看到菜品，才能加购和完成等待。"

### 4. 新增菜品（`#/merchant/dish/new`）
- 说明："烧烤店会显示对应的店内分类，外部分类仍由店铺类型决定。"
- 字段：
  - `菜品名称`（例："明天会后悔炸鸡桶"）
  - `菜品描述`（例："外酥里嫩，专治深夜空虚。"）
  - `价格（元）`
  - `热量估算（千卡，可选）` + 提示 "不填会按分类估算"
  - `菜品分类`（下拉，烧烤店的是：烤串/烤肉/烤海鲜/烤蔬菜/凉菜/主食/小吃/饮品/套餐/加料）
  - `菜品图片`（上传）
- 两个 CTA：`保存菜品` / `返回后台`

## 五、数据模型（推断）

```ts
// 后端应该有的核心表
type Shop = {
  id: string
  slug: string           // "late-bbq-lab" 类似
  code: string           // "NIGHT-ABC123" 找回凭证（唯一）
  name: string
  category: 'bbq' | 'fried' | 'tea' | 'dessert' | 'malatang' | 'burger' | 'noodle' | 'heavy' | 'mixed'
  desc: string
  cover: string          // 图 URL
  tags: string[]         // ["今晚免配送费", "炭火很到位"]
  rating: number         // 4.8
  monthSold: string      // "9999+" | "新店开张"
  avgPrice: number       // 42
  eta: [number, number]  // [28, 42] 分钟
  createdAt: number
  isNew: boolean         // 新店标签
  isSeed: boolean        // 是否官方种子店
}

type Dish = {
  id: string
  shopId: string
  name: string
  desc: string           // 情绪化文案
  price: number
  calories?: number
  subCategory: string    // 店内子分类
  image: string
  monthSold: number
}

type Order = {
  id: string             // "30822"
  shopId: string
  items: { dishId, name, qty, price }[]
  total: number
  address: string        // 从预置梗地址选一个
  couponName: '今晚不点券'
  remark: '少放后悔，多放一点清醒'
  etaSeconds: number     // 720-1200 之间随机
  createdAt: number
  riderName: string      // "小冷静" | "小理智" 随机 nickname
  // 无 status —— 永远只在 waiting 状态，没有 delivered
}

type ShopStats = {
  shopId: string
  visits: number     // 今日访问
  orders: number     // 今日下单
  virtualIncome: number
  cravedCount: number  // 被馋数（可能是加购次数）
}
```

**关键点**：
- 无用户账号系统；顾客端完全无登录
- 店家端只有 `店铺口令` 作为找回凭证，本质是**匿名店铺 + 密钥找回**
- 订单永远停在"配送中"，没有"已送达"状态
- 数据统计维度有点意思：加了个"被馋"指标

## 八、这是一个可以照抄的架构

**技术选型建议**（结合你想做 Web + 小程序双端）：
- **一套代码双端**：`uni-app` (Vue3) 或 `Taro` (React) 都行
- **纯前端 + 云函数/Serverless 存店家数据**：因为顾客端是完全预置的种子数据，只有店家端需要真持久化
- **状态**：Pinia / Zustand
- **样式**：CSS 变量 + 每端各自的样式适配层
- **假数据**：种子店 12 家 + 每家 8-15 道菜 + 一大堆情绪化文案（这是灵魂，得原创一份新的）

**功能优先级**（如果分期做）：
1. **P0（灵魂）**：首页 → 选店 → 菜单 → 结算 → 等待页（含倒计时+地图+三个安抚按钮）
2. **P1**：分类切换、地址梗多选、店家端开店 + 新增菜品 + 后台四大指标
3. **P2**：店铺口令找回、菜品图片上传、"被馋"事件埋点
4. **P3**：小程序端适配、分享卡片、每日一句安抚文案

## 九、一些好玩的细节（防止复刻时丢灵魂）

1. **"今晚不点券"**——不是"优惠券"，是**"不点"券**，暗示今晚别点，双关。
2. **地址列表全是梗**：明早不会后悔小区 / 被窝 3 号楼 / 理智恢复中心 / 我的深夜情绪收件处
3. **骑手名字**：小冷静、小理智（配合"稀释冲动"的主题）
4. **数据卡兜底文案**："今天还在等第一位客人"、"你的菜今天被馋了 0 次" —— 不冷冰冰
5. **等待页三个按钮**不是随便挑的动作："喝水 / 刷牙 / 我还是很想点"——**刷牙**这个反直觉动作是产品心理学：物理上刷完牙就不太想吃东西了
6. **菜品文案要写足**："开完会还没被夸，先给自己一点香气"——每道菜都是一句情绪句，情绪切入点比菜品本身重要
7. **"预计省下 ¥72 · 大约少摄入 2480-2880 千卡"** ——把"不点"的正反馈立刻反给用户

## 十、待你确认的问题

1. 你想仿的粒度：**像素级复刻** vs **借鉴产品概念做自己的版本**？
2. 目标端优先级：**Web 优先** 还是 **小程序优先**？两端一起做的话代码栈建议 uni-app。

4. 你有能力写文案吗？种子店铺 + 菜品的情绪文案（几百条）是这个产品的核心，AI 生成的和手写的差异巨大。
5. 项目名 / 品牌名想好了吗？（已经定了：COCO 的深夜食堂）
